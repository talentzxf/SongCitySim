# 038 — 道路 / 地形 / 建筑 交互规范（Interactions）

概述：将地形、道路与建筑作为高度耦合的三角关系来设计，保证每个系统的决策都会传导到其它两个系统，形成丰富策略而不是孤立系统。

要点细化：
- 影响矩阵（高层）
  - 地形 → 道路：高差、河流和沼泽限制路网布局，桥梁/堤坝为跨越要点
  - 地形 → 建筑：可建类型受地形影响（湿地限制工业/居住）；肥力影响农业类建筑输出
  - 道路 → 建筑：靠近主干道提高商业/市场价值；噪音降低住宅幸福度；道路等级决定物流效率
  - 建筑 → 道路：高密度建筑产生流量，促使道路升级或增加公共交通需求
- 连锁反馈与回路
  - 投资回路：投资道路→提高物流→市场活跃→税收增加→可再投资
  - 风险回路：在洪泛区建设→提升短期收益→遇洪灾损失→财政压力→无法修复
- 举例情境（供设计文档/平衡团队使用）
  - 沿河市场带来高游客但洪水季需临时撤摊；玩家可建堤或将商业撤到高地
  - 山城地形使房屋建造成本高，但房价与防灾值高，适合发展贵族/文化产业
  - 新建主干道贯穿作坊区，减少货运成本，但引入噪音导致附近民居幸福下降
- 预测与提示系统
  - 在建筑放置或道路铺设前，提示关键影响（如“此路段修建会将A区平均通勤时间下降 12%”）
  - 当地形改造将改变大量地块，弹出影响预览并估计短/中/长期收益/风险
- 规则冲突与优先级
  - 当多系统冲突（例如道路需要占用耕地），遵循：玩家选择＞法律/政策限制＞物理不可行
  - 提供溢出补救措施（如征用补偿/迁移补贴）作为政治/经济成本

优先级：P0（核心交互）

验收标准：
- 在多个示例地图中验证交互矩阵：至少 3 个场景能体现不同权衡（农业、沿海旅游、山城工业）
- UI 中在关键操作时提供明确的影响预览与建议方案

设计注意事项（PM 角度）：
- 交互系统要帮助玩家做出“可理解”的选择，避免隐藏的大数值惩罚导致玩家挫败
- 将复杂规则封装成可视化的提示与热图，降低学习成本

扩展想法：
- 可引入“法规/地方法令”系统，允许玩家以政策形式缓解或强化某些交互（如禁建令、补贴农田保全）
