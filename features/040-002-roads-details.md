# 002.x — 道路规划与交通系统细化（Roads & Traffic 细分）

目标：把道路与交通拆成可交付的表格、工具与仿真层级，便于 QA、平衡与美术对齐。

2.1 道路等级与属性表（必须）
- id / name / description
- base_cost_per_tile
- maintenance_per_day
- capacity_persons_per_minute
- capacity_goods_per_minute
- base_speed_modifier
- noise_level
- scenic_modifier (positive/negative)
- allowed_vehicle_types (pedestrian, cart, boat)
- upgrade_path (e.g., lane -> paved -> stone -> broadway)

2.2 路网规划工具与 UX（必须）
- 工具模式：自由画线、矩形/网格铺设、自动连通化（连接最近节点）
- 交叉口工具：自动生成交叉口/环形交叉或手动放置路灯/驿站
- 暂停施工/封路预览：展示施工期间的服务中断影响
- 模板存储：保存/加载路网模板（分享/模组使用）

2.3 交通仿真层级（必须）
- 低精度（默认移动端）：以路网容量与流量系数估算平均通行时间
- 中等精度：路段队列与简化路径搜索（减少密集路径计算）
- 高精度（可选 PC/服务器）：真实 agent 路径规划、优先规则与事故模拟
- 切换策略：在大地图或低电量模式自动降级仿真

2.4 升级、维护与政策（必须）
- 升级成本表与提升指标（容量/速度/噪音降低）
- 维护模型：道路磨损率、维修排队与预算优先级
- 政策工具：通行税、货运时段限制、道路收费、机动车限制

2.5 可视化与分析（必须）
- 拥堵热力图、平均通勤时间热图、主要物流路由高亮
- 路段统计界面：流量、平均等待、事故/维护记录
- 性能提示：推荐升级建议（例如“此路段建议拓宽或开驿站”）

2.6 示例场景与任务（建议）
- 商业带减速：缓解商业区拥堵并提升步行街美感
- 港口连通：优化码头-仓库-市场的货物路线以降低缺货风险
- 灾害应急：桥梁坍塌导致需快速绕行的临时道路建设任务

验收条件：
- 提供道路属性表与至少 3 套路网模板
- UI 能显示拥堵热力图并提出升级建议
- 能在不同仿真层级下运行并输出一致性指标（平均通勤/货物延迟）

备注（PM 指示）：
- 把复杂交通仿真封装为可理解的指标与建议，避免把玩家暴露在难以预测的数值中
- 移动端默认低精度仿真，需有清晰的降级提示与体验保证
