# 004.x — 核心模拟系统细化（Core Systems 细分）

目标：把核心模拟拆解为可交付的子系统、事件流与指标定义，保证各团队围绕明确接口并行工作。

4.1 模拟时钟与 Tick 系统（必须）
- tick_length_seconds（默认 1s 对应游戏内 1 分钟，可配置）
- 时间缩放控制：暂停、x0.5、x1、x2、x4
- 季节/年/月 日历系统（用于节日、农耕季节、洪水季）
- 存档点与快照频率（自动/手动）

4.2 人口模型（必须）
- 人口分类：年龄段（child/adult/elder）、职业（farmer/craftsman/merchant/official）、skill_level
- 人口流动：出生率、死亡率、迁入迁出率（受就业、住房、卫生影响）
- 家庭/户口模型（可选）：影响税收与住房分配
- 城市等级与人口上限：城市等级解锁更多地块与服务

4.3 需求与服务系统（必须）
- 需求种类：住房、就业、教育、医疗、娱乐、安全
- 服务覆盖模型：建筑提供服务半径、容量和排队机制
- 需求优先级与替代：当某服务不足，居民会有替代行为（如远距求医）

4.4 资源与物流（必须）
- 货物类型：粮食、布料、木材、陶瓷、金属、奢侈品
- 生产链规则：输入产出配方、加工时间、产量波动
- 库存与仓储：仓库容量、损耗、周转率
- 运输成本模型：与道路等级/距离/交通拥堵关联

4.5 财政与经济流（必须）
- 收入来源：税收、商税、港口费、旅游收入、贸易差价
- 支出项：维护、工资、项目投资、救灾拨款
- 预算周期（每 tick 汇总/每日结算）与财政预警阈值

4.6 指标与仪表盘（必须）
- 核心指标：人口、失业率、幸福度、财政余额、污染指数、犯罪率、旅游量
- 分层仪表盘：区级/市级视图、历史曲线、对比分析（期比/同比）

4.7 系统接口与扩展点（必须）
- 事件总线：定义标准事件（resource_changed, population_changed, disaster_triggered）
- 模块化服务：允许替换仿真精度、替换人口模型或物流算法
- 数据导出：CSV/JSON 用于分析与模组

验收条件：
- 定义并文档化每个子系统的输入/输出接口与关键事件
- 提供可运行的最简模拟（tick、人口波动、税收结算）作为集成验收样例

备注（PM 指示）：
- 保持子系统之间的低耦合高内聚，接口文档要先于实现完成
- 设定默认参数集以便策划快速迭代与测试
