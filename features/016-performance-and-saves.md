# 016 — 性能、存档与容错

概述：保障在 Web 与移动端的性能稳定，支持本地与云端存档、断网容错与回滚机制。

关键功能点：
- 性能目标：低端 Android 30fps 目标、主流设备 60fps；内存上限与GC策略
- LOD 与实体抽样：地图缩放时减少活跃实体数，使用聚合仿真加速
- 存档方案：本地存档（IndexedDB / 本地文件）+ 云存档（用户登录/服务器存储）
- 存档兼容性：版本迁移与回滚工具，保障新版本不会破坏旧存档
- 崩溃/异常收集：集成 Sentry/Firebase Crashlytics

优先级：P0（性能）、P1（云存档）

验收标准：
- 在指定测试机型上满足帧率与内存目标或有降级方案
- 存档能成功保存/加载并能处理版本升级迁移

实现要点：
- 采用数据驱动的保存格式（JSON 压缩/二进制可选）与差异化增量存档
- 性能分析工具链纳入 CI 流程（自动化基准测试）
