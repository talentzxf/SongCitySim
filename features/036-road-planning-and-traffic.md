# 036 — 道路规划与交通系统（Road Planning & Traffic）

概述：道路是连接城市功能的脉络。道路规划决定物流效率、市民通勤、土地价值与城市美观。设计一套既支持策略深度又可在性能上分级的交通系统。

目标：让道路成为城市设计的核心决策点：选路、升级、维护与公共交通策略都应影响城市运转效率与玩家反馈。

要点细化：
- 道路类型与属性
  - 等级/类型：小巷、街道、主干道、官道、商道、石板路、马车道
  - 专用道路：货运道、行人步道、河岸码头道、单行道、桥梁/隧道
  - 属性：建造成本、维护费用、容量（车辆/行人）、通行速度、景观/噪音影响
- 规划工具与 UX
  - 路网规划工具：线性铺设、批量铺设（沿线快速铺设）、网格/曲线模式
  - 交叉口编辑：十字路口、T 字、环形交叉；可设置灯控/驿站/集散广场
  - 拆迁/改路预览：变化对周边建筑/交通的影响预估（短期封路/长期通行影响）
  - 路网模板：玩家可保存/加载道路布局模板（供模组/分享）
- 交通仿真粒度（三档可选）
  - 抽象层（低）：以道路等级 + 流量系数计算平均通行时间（适合低端设备）
  - 中等层：分路段容量与车辆队列（有限路径搜索与拥堵传播）
  - 细粒度（高）：Agent-based 模拟，单个 NPC/货车使用路径规划（适合服务器或高端设备/多人场景）
  - 游戏可按性能设置切换模拟层级
- 交通影响与衍生系统
  - 时间成本：平均通勤时间影响幸福度与生产效率
  - 物流成本：货物运送时间影响市场价格与供应链效率
  - 污染与噪音：道路等级与流量影响周边环境指标、房价倒扣
  - 公共交通接入：驿站/渡口可建立路线减轻主干道负担并提供低成本运输
- 升级、维护与政策
  - 道路升级：扩大车道、铺设石板、加建桥梁，带来容量/速度提升与维护上升
  - 维修与耐久：道路破损率、维护周期与预算规划
  - 交通政策：车辆限制、货运时段、拥堵税、公共交通补贴
- 可视化与分析工具
  - 拥堵热力图、平均通勤时间地图、物流瓶颈点标注
  - 路段统计：流量、等待时间、事故率、维修需求
  - 路径演示模式：选中建筑后高亮显示典型出行/货物流动路线（流水线可视化）
- 游戏性与决策权衡
  - 设计 trade-off：建多条小路 vs 构建一条主干道；桥梁带来通行但高成本
  - 城市美观与商业价值：步道与广场提升文化/旅游但降低车流
  - 灾害与临时封路事件：桥梁损坏或道路沉降会迫使玩家临时重新规划

优先级：P0（必须与地形紧密结合）

验收标准：
- 玩家能使用道路工具构建基本路网并看到交通指标（拥堵、平均通勤）
- 至少实现一种拥堵可视化与一条升级道路（提升容量并改变通行时间）
- 支持在低/中/高三档仿真精度间切换并在不同档位下有合理表现

设计注意事项（PM 角度）：
- 将复杂仿真抽象为玩家可理解的指标与可视化，而非展示内部算法细节
- 维护玩家自由制定创造性的道路布局（鼓励桥、运河与街巷混搭）
- 与存档、性能与多平台发布策略协同，确保移动设备有轻量默认模拟层

扩展想法：
- 引入“交通大师”事件/任务（优化城市 3 个瓶颈或完成大型工程以获称号）
- 支持公共交通线路的运营优化（票价、班次、换乘枢纽设计）
