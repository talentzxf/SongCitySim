# 005.x — 存档、性能、UI 与新手引导的细化（Saves/Perf/UI/Onboarding 细分）

目标：将涉及 P0 项目的横向支撑系统拆细为具体交付清单，保证核心玩法在多平台与用户旅程中可用且稳定。

5.1 存档格式与同步（必须）
- 存档层次：快照（整档）、差分（增量改动）、日志回放（事件流）
- 同步冲突策略：last_write_wins / user_merge_prompt / server_resolution
- 存档大小目标：MVP ≤ 2MB（gzip），可增量扩展
- 加密与隐私：敏感字段加密、用户授权导出/删除接口

5.2 性能与 LOD 策略（必须）
- 实体 LOD：当缩放/距离改变时减少渲染与仿真实体
- 仿真抽样：对 NPC/作坊等使用采样仿真以降低 CPU
- 内存预算：设定低端设备中期运行内存上限并提供降级选项
- 离线/后台行为：当 app 后台或网页非活跃时降低仿真频率

5.3 UI 信息架构与可视化（必须）
- 仪表盘层级：全局 HUD、区级面板、建筑详情面板、事件弹窗
- 可视化图层开关：让玩家选择显示哪些图层（地形/洪水/拥堵/污染）
- 交互反馈规范：确认流程、撤销、施工队列可视化
- 本地化与文本长度预算：中文/英文/繁体应对文本溢出策略

5.4 新手引导分阶（必须）
- 引导阶段 1（0–15 分钟）：基础建造、税收、放置第一条道路
- 引导阶段 2（15–60 分钟）：人口增长、基本服务（医疗/教育）、小任务
- 引导阶段 3（1–4 小时）：政策、升级、灾害应对
- 教学 KPI：完成率、卡关点（每步触发失败率）、转化为付费的引导内购事件

5.5 测试与监控（必须）
- 性能基线测试：设定代表性设备列表与帧率/内存目标
- 存档兼容测试：跨版本加载/迁移测试用例
- 仪表盘埋点：关键 UI 操作埋点（打开地图、切换图层、触发升级）

验收条件：
- 给出存档格式规范与样例文件，并说明冲突处理流程
- 提供性能目标清单（低/中/高端设备）与降级方案
- 发布新手引导任务清单与教学埋点定义

备注（PM 指示）：
- 把兼容性与性能作为早期验收标准，避免核心玩法实现后遭遇不可接受的体验退化
- 新手引导应与产品指标绑定，持续迭代优化
