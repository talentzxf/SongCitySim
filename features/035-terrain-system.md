# 035 — 地形系统（Terrain System）

概述：地形是城市玩法的基础约束与玩法驱动力。需要设计地图生成、地形属性、可修改地形（造地/整治）、生态/洪水模型与与建筑/交通的交互规则。

目标：让地形既成为策略要素（选址/防灾/农业/景观），又不成为玩家的阻碍；提供可视化工具让玩家感知地形成本/收益/风险。

要点细化：
- 地形要素
  - 高程（海拔 / 等级）：影响水流、桥梁、土地价格、施工难度
  - 水系（河/湖/运河/海岸线）：决定通航节点、渔业、洪水风险与美观值
  - 土壤/肥力：决定农作物产出、作坊原料（如陶土）分布
  - 地表类型：耕地、湿地、沼泽、山地、平原、城市平整地
  - 资源节点：盐田、矿脉、泉眼、良田等可被开发的特殊地块
- 地形生成与编辑
  - 随机生成参数：河网密度、起伏程度、沿海/内陆分布、资源密度、历史遗迹分布
  - 预设地图与剧本地图支持（可直接还原历史或关卡设定）
  - 地形编辑器（玩家可用）：地形平整、填海、开渠、建堤、防洪工程（有成本/时间/生态影响）
- 与建筑/道路的交互
  - 建筑可建性规则：不同建筑/道路在特定地形上有允许/禁止/加成本状态
  - 地形改造成本：按体积/面积计费，影响财政与建设时间
  - 景观加成：河岸、公园、山景会带来旅游/房价加成
  - 洪水/排水系统：按地形坡度与排水系统计算淹水概率与影响范围
- 游戏性与策略
  - 选址权衡：沿河地价高但有洪水风险；高地防灾但道路成本高；肥沃平原适合农业但会吸引竞争土地使用
  - 长期改造策略：投资堤坝/引水渠可解锁大片可开发土地，但会降低生态指标并有维护费用
  - 资源与贸易：特殊资源点带来独特商品/产业，影响区域发展方向
- 可视化与玩家工具
  - 多层可视化图层：高程、洪水风险、肥力、建设可行性、土地价格热图
  - 预览模式：在进行地形改造或建筑前，显示改造后的地形、成本、影响指标（生态/旅游/防灾）
  - Undo/Redo 与施工计划队列
- 平衡与参数调优
  - 地形稀缺与区域价值：自然稀缺资源/平地会有溢价
  - 改造节奏：设置改造成本随城市规模/时代递增，避免玩家早期无限填海解法
  - 难度区分：初级地图平整、资源分布有利；高级地图地形复杂且灾害风险更高

优先级：P0（核心玩法）

验收标准：
- 地图生成器能输出 N 种不同特征的地图（至少：内陆平原、河网丰沛、山城、沿海）
- 玩家能在 UI 中看到并切换高程/洪水/肥力等图层
- 地形改造（堤坝/填海/挖渠）能改变建造可行性并消耗预算与时间

设计注意事项（PM 角度）：
- 让“地形决策”成为长期策略，而非单一最佳解；通过成本/维护/生态/社会指标形成多目标权衡
- 美术与地形数据要有明确的映射规则（如高程色带与实际建造高度）
- 与灾害、农业、旅游、道路系统紧密联动，确保地形改造不会破坏其它系统的可玩性

扩展想法：
- 引入地质学家/水利专家官员，作为政策或项目提高改造效率或降低风险
- 随时代进步解锁更高级的地形工程（如抽水站、人工港口、地下排水网络）
